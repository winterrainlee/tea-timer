# 차 한 잔의 시간 — Design v0.1

> 히스토리 스냅샷. 활성 설계 기준과 반영/거부 이력은 [DESIGN.md](../../DESIGN.md)를 본다.

작성일: 2026-06-14

## 1. 제품 방향

`차 한 잔의 시간`은 차를 좋아하는 사람이 폰 브라우저에서 바로 열어 쓰는 정적 웹앱이다. 핵심은 단순한 카운트다운이 아니라, 차가 우러나는 짧은 시간을 시각, 소리, 화면 변화로 감각화해 "지금 따라낼 때"를 자연스럽게 깨닫게 하는 것이다.

사용 환경은 모바일 세로 화면이며, 최소 기준은 iPhone 13 mini이다. 따라서 모든 주요 조작은 하단 엄지 영역에 있어야 하고, 화면을 오래 응시하지 않아도 완료 시점을 알 수 있어야 한다.

## 2. 디자인 원칙

1. **도구 앱처럼 정돈되되, 차 도구처럼 따뜻해야 한다.**  
   UI는 절제되고 밀도 있게 유지한다. 대신 중앙의 우림 도구, 탕색, 증기, 소리에서 감각적 분위기를 만든다.

2. **숫자는 보조이고, 감각 신호가 주역이다.**  
   카운트다운 숫자는 정확성을 주지만, 사용자가 실제로 느끼는 신호는 탕색, 잔량 링, 글로우, 증기, 예고음, 완료 플래시가 함께 만든다.

3. **차 우리는 종료는 0초가 아니라 분리/따르기다.**  
   차는 시간이 끝났다고 바로 끝나는 것이 아니다. 찻잎과 찻물을 분리하거나, 잔에 따라내거나, 티백/인퓨저를 꺼내는 행동까지가 우림의 완료다.

4. **고정 도구는 이미지 자산으로, 변하는 현상은 코드로 만든다.**  
   우림 도구 자체는 공예품처럼 보여야 한다. 복잡한 도구 형태를 인라인 SVG path만으로 억지로 그리기보다, 고정된 도구 이미지를 자산화하고 찻물, 찻잎, 증기, 링 같은 동적 요소만 SVG/Canvas로 합성한다.

## 3. 우림 도구 구성

도구는 5종으로 시작한다. 너무 많은 도구를 제공하기보다, 차를 우리는 방식과 완료 행동이 뚜렷하게 다른 대표 도구를 고른다.

| 도구 | 역할 | 탕색 가시성 | 완료 행동 |
|---|---|---|---|
| 티팟 | 서양식 폐쇄형 주전자 | 안 보임 | 잔/숙우에 따르기 |
| 동양식 다호 | 동양식 폐쇄형 소형 주전자 | 안 보임 | 숙우/찻잔에 빠르게 따르기 |
| 개완 | 뚜껑 있는 열린 우림 도구 | 보임 | 뚜껑 열고 따르기 |
| 머그 | 일상적 캐주얼 우림 | 윗면만 보임 | 마시기 또는 티백/인퓨저 꺼내기 |
| 표일배 | 버튼식 분리 도구 | 챔버 안에서 보임 | 버튼을 눌러 찻잎과 찻물 분리 |

이 5개는 형태가 아니라 **우림 방식**으로 나뉜다. 티팟과 동양식 다호는 모두 주전자이지만, 같은 그림으로 일반화하지 않는다. UX 모델에서는 같은 `closed-pot` 계열로 묶고, 시각 표현에서는 문화권, 재질, 실루엣으로 분리한다.

## 4. 도구 일반화 모델

도구 데이터는 다음 개념을 가져야 한다.

```js
{
  id: "eastern-pot",
  family: "closed-pot",
  culture: "gongfu",
  material: "zisha-clay",
  silhouette: "compact",
  liquorVisibility: "none",
  completionAction: "pour",
  requiresSeparationAction: false
}
```

핵심 필드는 다음과 같다.

| 필드 | 의미 |
|---|---|
| `family` | 도구의 기능적 계열. 예: `closed-pot`, `lidded-bowl`, `cup`, `separator` |
| `culture` | 시각/행동 리듬의 문화적 성격. 예: `western`, `gongfu`, `casual` |
| `material` | 재질. 예: `porcelain`, `clay`, `glass` |
| `silhouette` | 화면에서의 외형 특성. 예: `round-large`, `compact`, `tall`, `bowl` |
| `liquorVisibility` | 탕색을 어디까지 보여줄 수 있는가. 예: `none`, `top`, `bowl`, `chamber` |
| `completionAction` | 타이머 종료 후 실제 사용자가 해야 하는 행동 |
| `requiresSeparationAction` | 앱 안에서 추가 상태 전환 버튼이 필요한가 |

### 4.1 Closed Pot: 티팟과 동양식 다호

티팟과 동양식 다호는 모두 내부 탕색이 보이지 않는 폐쇄형 주전자다. 그래서 우림 진행은 내부 색 변화가 아니라 외부 글로우, 증기, 링, 소리로 전달한다.

티팟은 크고 여유 있는 도자 주전자다. 넓은 손잡이, 긴 주둥이, 둥근 몸통이 특징이다. 화면에서는 부드럽고 큰 글로우, 완만한 증기, 안정적인 실루엣이 어울린다.

동양식 다호는 작고 밀도 있는 다포용 주전자다. 티팟과 머그가 백색 도자 계열이므로, 다호는 **자사 점토의 적갈색 질감**으로 재질 다양성을 만든다. 낮고 둥근 몸통, 짧게 붙은 주둥이, 몸통과 자연스럽게 이어지는 손잡이, 작은 뚜껑, 무광에 가까운 흙 표면이 중요하다. 화면에서는 낮고 단정한 글로우, 짧고 집중된 증기, 빠른 반복 우림의 느낌이 어울린다.

참조 이미지 방향:

- 따뜻한 적갈색 또는 자주빛 갈색의 자사 점토.
- 작은 찻잔과 함께 놓였을 때 어울리는 스케일.
- 차판 위에 올라간 미니멀한 다구 느낌.
- 손잡이는 몸통에서 떨어진 고리가 아니라, 몸통에 붙어 이어지는 곡선 구조.
- 티팟/머그와 겹치는 백색 도자 질감은 피한다.

## 5. 완료 액션 철학

표일배만 특별히 버튼을 누르는 도구처럼 보이면 UX가 불균형해진다. 모든 도구에는 타이머 종료 후 사용자가 실제로 해야 하는 완료 행동이 있다.

| 도구 | 완료 메시지 방향 | 앱 내 상태 전환 |
|---|---|---|
| 티팟 | `따라내세요` | 별도 버튼 필수 아님 |
| 동양식 다호 | `바로 따라내세요` | 별도 버튼 필수 아님 |
| 개완 | `뚜껑 열고 따르세요` | 뚜껑 열림 연출 |
| 머그 | `꺼내세요` 또는 `드세요` | 모드에 따라 다름 |
| 표일배 | `버튼 눌러 내리세요` | 버튼 액션 후 아래 잔으로 배수 |

즉, 모든 도구는 `completionAction`을 갖는다. 다만 `requiresSeparationAction`은 표일배처럼 앱 화면에서 상태 변화가 명확히 필요한 도구에만 둔다.

## 6. 이미지 자산 + 동적 효과 레이어

우림 도구는 고정된 이미지 자산으로 만드는 것이 적합하다. 도구는 5개로 고정되어 있고, 공예품의 형태와 재질감을 코드 path만으로 충분히 표현하기 어렵기 때문이다.

권장 레이어 구조:

```text
stage
├─ vessel back image
├─ liquor SVG / leaves SVG / glow SVG
├─ vessel front image
├─ steam SVG or canvas
├─ timer ring SVG
└─ countdown text / completion message
```

### 6.1 불투명 도구

티팟, 동양식 다호, 머그는 한 장의 도구 이미지로 충분하다. 티팟과 동양식 다호는 내부가 보이지 않으므로 탕색 SVG는 쓰지 않고, 주변 글로우와 증기만 얹는다. 머그는 윗면 수면만 보조적으로 보여준다.

```text
closed-pot image
ambient glow
steam
timer ring
```

### 6.2 유리/반투명 도구

개완과 표일배는 유리 기물로 두고 내부 탕색이 보여야 하므로 앞/뒤 레이어를 분리하는 것이 최종 방향이다. 현재 구현은 단일 투명 PNG 위에 SVG 찻물/찻잎을 합성하되, 다음 단계에서 `back/front` 분리로 자연스러움을 높인다.

```text
glass vessel back image
liquor and leaves SVG
glass rim/front image
steam
timer ring
```

이 구조를 쓰면 도구 이미지는 고급스럽게 유지하면서도, 찻물이 차오르고 색이 짙어지고 찻잎이 펼쳐지는 효과를 계속 살릴 수 있다.

### 6.3 머그

머그는 내부 전체가 아니라 윗면만 보인다. 머그 이미지는 고정 자산으로 두고, 수면 타원과 증기만 SVG로 얹는다.

## 7. 자산 스타일 가이드

도구 이미지는 사진풍보다 앱 일러스트에 가까운 정물화 스타일이 좋다.

- 배경은 투명 PNG 또는 WebP.
- 화면 배경과 잘 섞이도록 어두운 차실 조명에 맞춘다.
- 외곽선은 너무 검지 않게 하고, 금빛 UI와 조화되는 따뜻한 하이라이트를 사용한다.
- 도구별 재질 차이가 분명해야 한다.
  - 도자: 부드럽고 은은한 반사
  - 동양식 다호: 자사 점토, 낮고 둥근 소형 실루엣
  - 유리: 얇은 림, 투명한 가장자리, 겹쳐 보이는 내부
- 그림자는 이미지 안에 강하게 넣지 않는다. 앱 레이어에서 공통 그림자를 줄 수 있어야 한다.
- 각 도구는 같은 캔버스 크기와 기준점으로 제작한다.

권장 파일 구조:

```text
/assets/vessels/
  teapot.png
  eastern-pot-zisha.png
  mug.png
  gaiwan-glass.png
  piaoyibei-glass.png
  # future split:
  gaiwan-glass-back.png
  gaiwan-glass-front.png
  piaoyibei-glass-back.png
  piaoyibei-glass-front.png
```

## 8. 구현 우선순위

1. **완료** — 현재 인라인 SVG 도구 본체를 이미지 레이어 구조로 분리한다.
2. **완료** — `VESSELS` 데이터에 `family`, `material`, `liquorVisibility`, `completionAction`, `requiresSeparationAction`, `asset` 필드를 추가한다.
3. **부분 완료** — 개완/표일배는 현재 단일 PNG 위에 찻물/찻잎 SVG를 얹는다. 다음 단계에서 `back/front` 이미지 분리를 적용하면 더 자연스럽다.
4. **완료** — 티팟/동양식 다호는 이미지 + 글로우 + 증기 + 링으로 단순화한다.
5. **완료** — 표일배만 앱 내 `버튼 눌러 내리기` 상태 전환을 유지한다.
6. **완료** — 완료 메시지를 도구별 `completionAction`에서 파생한다.
7. **완료** — iPhone 13 mini 홈 화면 모드의 세로 공간을 확보하기 위해 하단 도구 선택 줄을 없애고, 중앙 기물 양옆 화살표와 기물 이름만으로 도구를 바꾼다.
8. **추후** — 개인용 최소 기준은 iPhone 13 mini 세로 화면으로 유지하되, 배포 품질을 올릴 때 작은/짧은 화면과 큰 화면 대응을 점검한다. `100dvh`, 짧은 화면용 `@media (max-height: 700px)`, 세로 스크롤 fallback, 가로모드 안내 또는 레이아웃 축소를 검토한다.

## 9. 현재 판단

5개 도구 구성은 유지한다. 다만 도구 본체를 전부 SVG path로 구현하는 방향은 폐기하거나 최소화한다. 이 앱의 감각적 품질은 중앙 도구의 인상에 크게 좌우되므로, 고정 도구는 이미지 자산으로 만들고 동적 우림 현상만 코드로 합성하는 하이브리드 방식이 v0.1 이후의 기준이다.
