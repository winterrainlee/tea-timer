# 차 한 잔의 시간 · Time for Tea — 모바일 웹앱 설계 문서

> 베이스: `gaiwan-oolong.html` (단일 파일 SVG+Canvas 우롱차 타이머)
> 목표: 모바일에서 홈 화면에 추가해 쓰는 PWA로 재설계. 여러 차 프리셋 지원, 수동 회차(多泡), "이제 그만 우려라"는 깨달음을 시각·청각·화면으로 유도.
> 작성일: 2026-06-14

> 최신 도구/자산 철학은 `design-v0.1.md`를 기준으로 한다. 특히 기존 `자사호` 방향은 백자 계열 **동양식 다호**로 확장/변경되었고, 도구 본체는 이미지 자산 + 동적 SVG 효과 레이어 방식으로 전환한다.

---

## 1. 제품 개요

### 1.1 한 줄 정의
끓는 물을 부은 뒤 **수십 초의 짧은 시간** 동안, 화면을 보든 안 보든 "**지금 따라낼 때**"임을 자연스럽게 깨닫게 해 주는 차 우림 타이머.

### 1.2 핵심 사용자 흐름 (Happy Path)
1. 앱을 연다 (홈 화면 아이콘 → 전체화면 실행).
2. **차를 고른다** (프리셋 칩/바텀시트). → 권장 시간·온도가 슬라이더와 안내에 자동 반영.
3. **슬라이더로 시간을 미세 조정**한다 (기본값에서 ±).
4. **시작**을 누른다 → 물 붓는 연출 → 우림 시작.
5. 시간이 흐르며 **찻물이 차오르고 색이 짙어진다**. 막바지에 **임박 신호**가 강해진다.
6. 완료 **5초 전 "준비" 예고음** → 완료 시 **종소리 + 화면 플래시 + 뚜껑 열림 + "지금 따라내세요"**.
7. 다음 우림이면 **다음 泡** 버튼 → 회차가 1 증가, 시간만 다시 조정해 반복.

### 1.3 설계 원칙
- **앰비언트 우선**: 숫자를 안 봐도 "거의 다 됐다"가 주변시(peripheral vision)로 느껴져야 한다.
- **다중 감각 중복**: 색만으로 알리지 않는다. 시각 + 소리 + 화면 밝기를 겹쳐 어느 하나를 놓쳐도 전달된다.
- **수십 초 스케일에 맞춘 긴장 곡선**: 30초짜리 타이머는 30분짜리와 다른 리듬이 필요하다. 마지막 ~10초에 집중적으로 긴장을 끌어올린다.
- **엄지 영역 우선**: 모든 주요 조작은 화면 하단 1/3(엄지 도달 범위)에 둔다.
- **오프라인·즉시 실행**: 네트워크 없이도 1초 안에 뜨고 동작한다.

## 2. 베이스 코드 분석 (`gaiwan-oolong.html`)

### 2.1 구조
| 레이어 | 기술 | 역할 |
|---|---|---|
| 개완 본체 | 인라인 `<svg viewBox="0 0 800 900">` | 유리잔·찻물·찻잎·뚜껑(蓋)을 벡터로 렌더 |
| 파티클 효과 | `<canvas>` 위 2D | 물줄기, 물방울(splash), 잔물결(ripple), 김(steam/wisp) |
| 합성 사운드 | WebAudio (`OscillatorNode`) | 외부 파일 없이 종소리 합성 (E5 + B5) |
| 폰트 | Google Fonts | Noto Serif KR / SC, Space Mono |

### 2.2 상태 머신 (현재)
```
idle ──startPour()──▶ pour(2.6s 물붓기) ──enterSteep()──▶ steep(우림) ──finishSteep()──▶ done
  ▲                                                                                          │
  └──────────────────────────────── reset() ◀───────────────────── "다음 泡"(infusion++) ────┘
```
- `infusion`(泡 차수)이 회차별 농도(`peakAlpha[]`)·찻잎 펼침(`openTgt[]`)을 결정 → **공부차식 多泡이 이미 구현됨**.
- 우림 중 `liquorFill` 투명도가 `easeInOut`으로 차오르고, 찻잎이 펼쳐지고, 김이 사인 곡선으로 점점 자주 핀다.
- `prefers-reduced-motion` 분기 존재(연출 생략).

### 2.3 모바일 관점의 한계 (재설계 포인트)
1. **터치 타깃 부족**: 슬라이더 thumb 15px, 컨트롤이 `position:absolute`로 하단에 겹쳐 배치 → 작은 화면에서 좁고 가림.
2. **"안 보면 놓침"**: 핵심 신호가 색 변화 하나라 화면을 안 보면 타이밍을 놓친다. 소리는 완료 순간뿐, **예고가 없다**.
3. **단일 차 고정**: 색·시간·온도가 청향 우롱 하나에 하드코딩.
4. **모바일 실사용 미대응**: 화면 자동 꺼짐(차 우리는 동안 화면이 꺼지면 신호 못 봄), 백그라운드 탭에서 `requestAnimationFrame` 정지 시 카운트다운·완료 누락, 더블탭 줌, 노치 안전영역(safe-area).
5. **임박 긴장 부재**: 색이 처음부터 끝까지 매끄럽게만 차올라 "이제 그만"이라는 **변곡점의 느낌**이 약하다.

### 2.4 보존할 자산 (재사용)
- SVG 개완 + Canvas 파티클의 **2중 레이어 렌더 구조** → 그대로 계승, 색만 데이터 주도로 일반화.
- WebAudio **합성 종소리** → 외부 파일 불필요 = 오프라인 PWA에 이상적. 예고음·틱을 같은 방식으로 확장.
- `infusion` 기반 **회차 모델** → 수동 회차로 유지.

## 3. UI/UX 재설계 (모바일 세로)

### 3.1 화면 레이아웃 (단일 화면, 세로 고정)
```
┌─────────────────────────────┐  ← safe-area top inset
│   清香烏龍 · 청향 우롱        │   ① 헤더: 차 이름 + 회차(第二泡)
│        第二泡                 │
│                             │
│        ╭─────────╮          │   ② 히어로: 개완 SVG
│      ╱  (개완 그림) ╲        │      + 시간 잔량 링(원형 게이지)
│     │   찻물 차오름   │       │      + 중앙 큰 카운트다운
│      ╲   0:18      ╱        │
│        ╰─────────╯          │
│                             │
│  [티팟 다호 ◉개완 머그 표일배]│  ⓪ 우림 도구 선택(아이콘 세그먼트)
│   ‹  ●청향 우롱  › ····      │   ③ 차 프리셋 (좌우로 넘겨 선택)
│                             │
│   우림 시간   [───●─────] 20초│   ④ 시간 슬라이더 + 미세조정(−/+)
│                             │
│   ┌───────────────────────┐ │   ⑤ 주 액션 버튼 (풀폭, 큼)
│   │      우리기 시작        │ │
│   └───────────────────────┘ │
│   처음부터        🔔 ⚙       │   ⑥ 보조: 리셋 / 음소거 / 설정
└─────────────────────────────┘  ← safe-area bottom inset
```

- **세로 고정**(`orientation: portrait`). 가로는 우선순위 밖.
- 상단 1/2은 **감상 영역**(개완+링+숫자), 하단 1/2은 **조작 영역**(엄지 존).
- 모든 탭 타깃 최소 **44×44 CSS px**. 슬라이더 thumb 시각 28px / 히트영역 44px.

### 3.2 컴포넌트별 재설계

#### ⓪ 우림 도구(vessel) 선택
- 5종 대표 도구를 작은 **아이콘 세그먼트**로 선택 → 히어로 SVG가 해당 도구로 교체됨: **서양 티팟 · 동양식 다호 · 개완(蓋碗) · 머그컵 · 표일배(飄逸杯)**.
- 도구는 차보다 덜 자주 바꾸는 "설정성" 선택 → 차 캐러셀보다 작은 아이콘 행으로 두고, 마지막 선택을 localStorage에 저장해 복원.
- **재질·투명도가 핵심 색 신호를 좌우**한다(§4.5). 요약:

| 도구 | 재질 | 탕색 가시성 | 형태 특징 |
|---|---|---|---|
| 개완 蓋碗 | 유리 | **보임** | 사발+뚜껑+받침, 뚜껑 들어 따름 |
| 표일배 飄逸杯 | 내열유리 | **보임** | 버튼식 거름 + 속거름망(찻잎 분리) |
| 서양 티팟 | 도자 | 안 보임 | 둥근 몸체+주둥이+손잡이+돔뚜껑 |
| 동양식 다호 | 백자/도자 | 안 보임 | 작고 둥근 소형 다호, 짧은 주둥이+몸통에 붙은 손잡이 |
| 머그컵 | 도자 | 윗면만 | 원통+손잡이, 뚜껑 없음 |

#### ① 헤더 — 차 이름 + 회차
- 큰 글자: 한글 차 이름(주), CJK는 장식적 보조. 회차는 `第二泡` + 한글 보조 `(2번째 우림)`.
- `aria-live`로 회차 변경을 읽어 줌.

#### ② 히어로 — 개완 + 시간 잔량 링 + 카운트다운 (핵심 재설계)
세 가지 정보를 **역할 분리**해 겹쳐 표현한다.

| 요소 | 의미하는 것 | 표현 |
|---|---|---|
| 개완 속 찻물 색·채움 | **차의 농도/품질** (얼마나 우러났나) | 색이 옅음→짙음, 찻잎 펼침, 김 |
| 개완을 감싸는 원형 링 | **남은 시간** (얼마나 남았나) | 평소엔 **가는 저대비 헤어라인**(개완에 시선 양보), 막바지에만 밝아지는 **줄어드는** 게이지 |
| 중앙 숫자 | **정확한 잔여 초** | `0:18` 큰 tabular 숫자 |

> 찻물은 "차오르는"(많을수록 진행) 직관, 링은 "줄어드는"(소진) 직관 → 둘을 합쳐 진행/잔여를 동시에 느끼게 한다.
>
> **링 시선 원칙** — 굵은 골드 풀링은 시간이 가장 많이 남은 시작 순간에 시선을 과하게 뺏는다(주인공이어야 할 개완을 가림). 그래서 링은 **기본적으로 가는 저대비 헤어라인(2px, 낮은 불투명도, 무채색에 가까운 골드)** 으로 두어 개완·찻물에 시선을 양보하고, **임박 구간(마지막 ~10초)에서만 밝기·굵기를 끌어올려** "남은 시간" 기능을 되살린다. 색상(hue)은 바꾸지 않고 **밝기/불투명도만** 변화시켜 경보처럼 튀지 않게 한다.

#### ③ 차 프리셋 선택 (캐러셀)
- **좌우로 넘겨 하나를 고르는 캐러셀**: 가운데 차가 선택 상태(골드 강조 + 탕색 점), 양옆에 이웃 차가 흐리게 미리보기, `‹ ›` 셰브런과 **페이지 점(dots)** 으로 "여러 프리셋 중 하나"임을 전달. 스와이프/탭으로 전환.
- 차 이름은 **줄바꿈 금지(`white-space:nowrap`)** — "청향 우롱" 같은 이름이 중간에서 끊기지 않게.
- 선택 시 슬라이더 기본값이 그 차의 권장 시간으로 갱신.
- **권장 물 온도는 차 선택 순간 토스트로만** 잠깐 표시(`권장 95°C` — 3초 후 사라짐). 상시 노출하지 않아 화면을 깔끔하게 유지.

#### ④ 시간 슬라이더 + 미세 조정
- 풀폭 슬라이더, 큰 thumb. **거친 조정**은 슬라이더(step 5초), **미세 조정**은 좌우 `−`/`+` 버튼(±1초).
- 잔여 숫자 영역을 탭하면 ±1초 nudge.
- 범위: 5–120초 (프리셋이 기본값 세팅).

#### ⑤ 주 액션 버튼 (상태별 라벨)
| 상태 | 라벨 |
|---|---|
| idle | `우리기 시작` |
| pour | `물 붓는 중…` (비활성) |
| steep | `우리는 중…` (비활성, 길게 누르면 "지금 멈추기" 옵션) |
| done | `다음 泡 우리기` |

- 비활성 시에도 진행 상황을 라벨로 전달.

#### ⑥ 보조 컨트롤
- `처음부터`(리셋), 음소거 토글, 설정(소리/햅틱/화면유지/언어).

### 3.3 모바일 인터랙션 디테일
- `touch-action: manipulation`, `user-scalable=no`로 더블탭 줌·핀치 방지.
- `env(safe-area-inset-*)`로 노치/홈바 회피 패딩.
- 슬라이더 드래그 중 햅틱(지원 시) 가벼운 틱.
- 첫 사용자 제스처(시작 버튼)에서 WebAudio 언락(`ensureAudio`) — 현재 코드 패턴 유지.

## 4. 완료 임박 신호 시스템 (제품의 심장)

확정된 신호 채널: **시각 + 소리(예고음·종소리) + 화면 켜둠·플래시**. (햅틱은 §9 향후 옵션)

### 4.1 우림 긴장 곡선 (steep 타임라인)
`t = 경과/총시간`. 마지막 ~10초에 긴장을 집중한다.

| 구간 | t 범위 | 시각 | 링 | 카운트다운 | 소리 |
|---|---|---|---|---|---|
| **A. 우러나기** | 0 ~ 0.5 | 색이 맑음→목표 색으로 서서히, 찻잎 펴지기 시작, 잔잔한 김 | 가는 헤어라인 · 거의 안 보임 | 차분한 흐린 색 | 무음 |
| **B. 무르익음** | 0.5 ~ 0.85 | 색이 목표치에 근접, 찻잎 활짝, 김 최고조, 잔 주변 따뜻한 발광↑ | 여전히 가늘게 · 천천히 감소 | 또렷해짐 | 무음 |
| **C. 임박** | 0.85 ~ 1.0 | **잔을 감싸는 헤일로가 맥동(pulse)**, 배경 글로우가 심장박동처럼 밝아짐, 숫자 확대·매초 펄스 | 밝기↑·살짝 굵어져 또렷 | 커지고 1초마다 톡톡 | **남은 5초: 예고음** + 매초 틱(옵션) |
| **완료** | t ≥ 1 | 뚜껑 열림(蓋 들림) + **전체화면 소프트 플래시** + 색 고정(이상 농도) + "지금 따라내세요 · 出湯" | 옅은 잔상으로 사라짐 | `0:00` 강조 | **완료 종소리** |

> 핵심 발상 — **설정 시간 = 이상점(sweet spot)**. 다가갈수록 시각이 "이상"으로 수렴해 완료 순간 색·찻잎·김이 가장 무르익은 상태로 고정된다. (과침 카운트업은 미도입 — 단순함 우선. 완료 후엔 "지금 따라내세요"만 유지.)

### 4.2 소리 설계 (WebAudio 합성, 오프라인)
모든 소리는 외부 파일 없이 합성. **정확도를 위해 우림 시작 시점에 audioContext 클록 기준 절대 시각으로 예약**한다(아래 §6.3).

| 사운드 | 시점 | 음색 | 목적 |
|---|---|---|---|
| **예고음 (준비)** | 완료 **5초 전** (단, **짧은 우림은 비례 축소**, 최소 **3초 전**) | 부드러운 단음(예: G5) 1회, 작은 게인 | "곧 따라낼 준비를 하라" |
| **카운트 틱** | 예고음 시점부터 1초마다 (5·4·3·2·1 또는 3·2·1) | 아주 짧은 우드블록 류 틱 | 초읽기 체감 |
| **완료 종소리** | 완료 | 기존 E5+B5 종소리(배음 4개) | 우림 끝 |
| 확인 블립 | 음소거 해제 시 | 880Hz 짧은음 | 상태 확인(기존) |

- 예고음 비례 규칙(안): `예고초 = clamp(round(총시간 × 0.2), 3, 5)`. 예) 12초 → 3초 전, 25초 → 5초 전.
- 틱은 예고음과 같은 구간(예고초→0)에 매초 1회. **기본: 예고음 ON, 틱 ON.** 음소거 시 전체 묵음.

### 4.3 화면 켜둠 · 플래시
- **Wake Lock**: `pour`+`steep` 동안 `navigator.wakeLock.request('screen')`로 화면 유지. `visibilitychange`에서 재획득, `done`/`idle`에서 해제. 미지원 브라우저는 graceful degrade(설명 토스트).
- **완료 플래시**: 전체화면 오버레이가 한 번 밝게 번쩍였다 사라짐. `prefers-reduced-motion`이면 번쩍임 대신 부드러운 페이드.
- iOS 등 일부 환경에서 PWA·웹앱의 화면 유지/오디오 제약이 있어 **시각+소리+화면을 중복**시켜 어느 하나가 막혀도 전달되게 한다.

### 4.4 접근성 안전장치
- 색에만 의존하지 않음(소리·숫자·플래시 중복).
- 카운트다운 `aria-live="polite"`(마지막 구간만 갱신 빈도↑), 완료 메시지 `aria-live="assertive"`.
- `prefers-reduced-motion`: 맥동/플래시/파티클 축소, 색·숫자·소리는 유지.

### 4.5 도구 투명도별 신호 적응
도구마다 "잔 속이 보이는 정도"가 달라 §4.1의 **색 곡선**을 그대로 쓸 수 없다.
- **투명(유리: 개완·표일배)** — 찻물 색·채움·찻잎이 보임 → §4.1 색 곡선 그대로.
  - **표일배 특이 구조**: 우림 중에는 **윗 거름망 챔버 안에 찻잎+물이 함께** 들어 있어 색이 거기서 우러난다(아래 잔은 비어 있음). **완료 시 버튼을 눌러야** 물이 아래 잔으로 내려가고 찻잎은 위에 남는다 → 완료 제스처 = **버튼 눌러 내리기**. 따라서 진행 색 변화는 **상단 챔버**에서, "이제 그만"은 **버튼 눌러 분리**로 표현.
- **불투명(도자: 서양 티팟·동양식 다호)** — 잔 속이 안 보여 "색 짙어짐" 단서가 사라짐. 대체:
  - 도구를 **감싸는 따뜻한 앰비언트 글로우**가 시간에 따라 차오르고 막바지에 맥동.
  - **김(steam)** 을 주둥이/뚜껑 틈에서 피워 진행 암시.
- **윗면만(머그)** — 입구의 **수면 색**만 시간에 따라 짙어지게.
- 헤어라인 링·숫자·예고음·틱·플래시·Wake Lock은 **모든 도구 공통**(이미 색 비의존) → 어떤 도구든 "이제 그만"은 전달됨.
- **세로로 긴 도구(표일배·머그)** 는 원형 링이 실루엣을 어색하게 감싸므로 **숫자 중심**으로 잔여를 표현(링은 생략하거나 슬림 바). 둥근 도구(개완·동양식 다호·티팟)는 헤어라인 링 사용.
- **완료 제스처도 도구별로**: 개완=뚜껑 들림(開蓋), 티팟/동양식 다호=살짝 기울여 따르는 힌트, 표일배=버튼 눌림(거름 내려감), 머그=완료 발광.

### 5.1 차 프리셋 (내장 기본값)

|---|---|---|---|---|---|---|
| 녹차 | 綠茶 | 80°C | 15초 | +10초 | `#d6dd86` | `#b9c45a` |
| 백차 | 白茶 | 90°C | 30초 | +10초 | `#ecdca0` | `#d8b46e` |

| 홍차 | 紅茶 | 90°C | 12초 | +5초 | `#c4622a` | `#9c3a18` |

*증가값은 "수동" 모드에서 다음 泡의 **제안 기본값**으로만 쓰임(자동 강제 아님).*
*탕색 2색은 SVG `liquor` 그라디언트의 `light/strong`에 주입 — 우림 초반엔 옅음, 농도가 오르며 짙음으로 수렴.*

> **짙은 차 주의** — 숙보이·홍차처럼 탕색이 거의 불투명하면 "농도가 차오르는" 시각 변화가 빨리 포화돼 시간 흐름이 색만으로는 잘 안 읽힌다. 이런 차는 색보다 **찻잎 펼침·김·헤어라인 링·숫자**에 우림 진행을 더 싣는다.

### 5.2 프리셋 객체 스키마
```js
{
  id: "oolong-light",
  name: { ko: "청향 우롱", cjk: "清香烏龍" },
  tempC: 95,                 // 안내 표시용
  baseSec: 20,               // 1泡 기본 시간
  suggestAddSec: 5,          // 다음 泡 제안 증가량(수동)
  liquor: {                  // SVG liquor 그라디언트에 주입
    light: [222,210,132],    // 옅을 때 RGB
    strong:[198,176,86]      // 짙을 때 RGB
  },
  infusionPeaks: [0,.42,.66,.78,.70,.58,.46,.38,.30], // 회차별 농도(기존 peakAlpha 일반화)
  note: "향이 가벼운 청향. 1~2泡은 짧게."
}
```
> 베이스 코드의 `liqLight/liqStrong`, `peakAlpha[]`, `openTgt[]`를 프리셋 필드로 일반화하면 렌더 코드는 그대로 두고 데이터만 바꿔 다(多)차를 지원한다.

## 6. 기술 아키텍처 (단일 HTML PWA)

### 6.1 파일 구성
"단일 HTML" 정신은 유지하되, **설치 가능 + 오프라인**을 위해 PWA 필수 파일을 최소로 추가한다(서비스워커는 별도 파일이어야 함 — 인라인 불가).
```
/index.html              # 앱 전체(마크업+CSS+JS) — 베이스 계승
/manifest.webmanifest    # 설치 메타
/sw.js                   # 서비스워커(앱 셸 캐시)
/icons/icon-192.png, icon-512.png, icon-maskable.png
/fonts/…                 # 폰트 셀프호스팅(서브셋) — 오프라인·속도용 (권장)
```

### 6.2 PWA 세부
- **manifest**: `name "차 한 잔의 시간"`(부제 "Time for Tea"), `short_name "차 한 잔"`, `display: "standalone"`, `orientation: "portrait"`, `theme_color/background_color: #120d09`, `lang: "ko"`, maskable 아이콘.
- **service worker**: 앱 셸 **cache-first**(버전 태그). 업데이트 시 새 캐시로 교체. 폰트 포함.
- **폰트**: Google Fonts 의존을 끊고 **셀프호스팅 서브셋**(사용 글리프만) → 오프라인 보장 + 초기 로드 단축. 미사용 시 시스템 serif 폴백.
- **설치 유도**: `beforeinstallprompt` 가로채 "홈 화면에 추가" 안내(선택).

### 6.3 타이머 정확도 (모바일 백그라운드 대응)
모바일에서 `requestAnimationFrame`은 탭이 가려지면 멈춰 카운트다운·완료가 틀어진다. 대응:
- **절대 시각 기준**: 시작 시 `endAt = performance.now() + steepMs` 저장. 매 프레임 잔여 = `endAt − now` 재계산(rAF가 끊겨도 복귀 시 정확).
- **오디오는 클록 예약**: `chime`과 `예고음`을 우림 시작 시 `audioCtx.currentTime + (steepMs/1000)` / `− 5`에 `setValueAtTime`으로 예약 → rAF 잔떨림과 무관하게 정확히 울림.
- **복귀 보정**: `visibilitychange→visible`에서 잔여 재계산. 가려진 동안 종료시각을 지났으면 즉시 `done`으로 점프해 완료 처리(베스트에포트).
- **화면 유지가 1차 방어선**: Wake Lock으로 화면을 켜 두면 앱이 포그라운드로 유지돼 위 문제 발생 자체가 줄어듦.
- (선택) Web Worker `setInterval` 틱으로 로직을 구동하면 백그라운드 throttle에 더 강함.

### 6.4 렌더 성능
- `DPR` 상한 2(기존 유지), 파티클 수 상한, 저사양 기기에서 `prefers-reduced-motion`/배터리 절약 시 파티클·맥동 감축.
- Canvas 리사이즈는 `resize`/`orientationchange` 디바운스.

## 7. 상태 머신 (재정의)

```
        ┌───────────────────────────── reset ───────────────────────────┐
        ▼                                                                │
   ┌─────────┐  start   ┌────────┐ pour끝  ┌────────────────────┐  t≥1  ┌──────┐
   │  idle   │ ───────▶ │  pour  │ ──────▶ │       steep        │ ────▶ │ done │
   │ 차/시간 │          │ 물붓기 │         │  A → B → C(임박)    │       │ 蓋열림│
   │  선택   │          │ 2.6s   │         │  ·5초전 예고음 예약 │       │ 종+플래시│
   └─────────┘          └────────┘         │  ·Wake Lock 유지   │       └──┬───┘
                                           └────────────────────┘          │ "다음 泡"
                                                                   infusion++ │ (시간 재조정)
                                                                            └──▶ pour …
```
- `idle`/`done`에서만 차 변경·슬라이더 활성. `pour`/`steep` 중 비활성(길게 눌러 중단만).
- 과침 카운트업은 `done`의 부가 표시(상태 전이 아님).

## 8. 접근성 · 다국어

- **언어**: 한국어 1차. CJK(`泡`, `出湯`)는 장식 보조. 설정에서 CJK 표기 on/off.
- **스크린리더**: 모든 버튼 한국어 `aria-label`, 카운트다운/완료 `aria-live`, 개완 SVG에 설명 `aria-label`(기존 보유).
- **모션 민감**: `prefers-reduced-motion` 존중(맥동·플래시·파티클 축소, 핵심 신호 유지).
- **대비**: 텍스트/배경 대비 WCAG AA 목표. 골드 텍스트 대비 점검.
- **터치**: 최소 44px 타깃, 슬라이더 키보드 조작(`focus-visible` 유지).

## 9. 향후 확장 (이번 범위 밖, 후보)

- **햅틱(진동)**: 임박/완료 시 `navigator.vibrate` 패턴. iOS Safari 제약으로 이번엔 보류 — 설정 토글로 추후 추가.
- **자동 회차 시간 제안 강화**: 누적 향/농도 학습 기반.
- **차 커스텀 등록**: 사용자 정의 차(이름·시간·색) 추가·저장.
- **우림 기록**: 오늘 몇 泡까지 우렸는지 세션 로그.
- **다국어 확장**: 영어/중국어/일본어.

> 구현 메모 — 단일 파일 `index.html`(바닐라 JS, SVG 렌더, WebAudio 합성음). 개완은 찻물 차오름·색 심화·찻잎 펼침·뚜껑 열림 전체 애니메이션, 표일배는 **윗 챔버 우림→메인 버튼 눌러 아래로 배수**, 불투명 도구(티팟·동양식 다호)는 **앰비언트 글로우+김**, 머그는 수면색. 세로 도구(표일배·머그)는 링 생략·숫자 중심. **개발 시 주의**: `sw.js`가 cache-first라 코드 수정 후엔 SW 등록 해제+캐시 삭제 후 새로고침해야 반영됨(아이콘 PNG는 아직 없어 설치 아이콘은 추가 필요).
